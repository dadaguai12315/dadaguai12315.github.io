Bspwm Proxy Config

 大大怪 收录于 Arch Linux
2023-10-06 2023-10-06 约 418 字 预计阅读 1 分钟 

问题描述

在bspwm这种窗口管理器中设置代理是非常麻烦的，但是使用clash for windows需要设置系统代理，在终端很难实现系统代理，在Arch wiki中可以找到解决方法:

export http_proxy=http://10.203.0.1:5187/
export https_proxy=$http_proxy
export ftp_proxy=$http_proxy
export rsync_proxy=$http_proxy
export no_proxy="localhost,127.0.0.1,localaddress,.localdomain.com"

如果代理环境变量要提供给所有的用户和所有的应用程序，上述的导出命令可以添加到一个脚本中，比如说在 /etc/profile.d/中的proxy.sh . 脚本必须被设置成可执行的executable. 当使用像Xfce这样不提供代理配置选项的桌面环境时，这种方法很有帮助。例如，Chromium浏览器将在运行XFCE时使用此方法设置的变量。

也就是说，在/etc/profile.d/目录下创建一个proxy.sh脚本（注意修改上述端口，一般为7890），然后赋予可执行权限，在登陆之后执行脚本即可设置系统代理。

sudo chmod +x proxy.sh
